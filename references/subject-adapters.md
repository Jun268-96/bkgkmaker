# Subject adapter checks

Read only the relevant row family while drafting and scope-reviewing. Record the selected `discipline_lens`, `discipline_pass`, and a concrete reason in the scope packet.

| Family | Subjects | Required semantic checks |
|---|---|---|
| Language | 국어, 영어 | The answer follows from the supplied passage, expression condition, language form, or media evidence. Do not make outside knowledge the hidden key. Use original or licensed text. |
| Mathematics | 수학 | The selected topic profile, representation, structural feature, units, exact calculation, and prerequisite boundary agree. Recompute every numeric answer. |
| Social | 사회 | Separate fact from judgment, state the comparison or spatial attribute, support causal or perspective claims with evidence, and refresh dynamic civic or geographic facts. |
| Moral | 도덕 | Supply a concrete situation and a reasoned criterion. Do not turn a debatable value claim into an unsupported context-free fact. |
| Science | 과학 | Separate observation, inference, and prediction. Do not imply an experiment occurred unless observations or data are supplied. Check procedure, variables, models, and safety when relevant. |
| Practical arts | 실과 | Check safe procedure, sequence, resource constraints, design choice, digital ethics, and any current safety guidance. |
| Performance | 체육, 음악, 미술 | Use quizzes only for supporting concepts, observation, process choices, or reflection. Do not claim factual recall replaces performance or creation. |
| Integrated | 바른 생활, 슬기로운 생활, 즐거운 생활, 통합교과 | Keep reading load age-appropriate and connect knowledge to daily life, inquiry, play, expression, or reflection without reducing the subject to recall. |
| Experiential | 창의적 체험활동 | Require the school's supplied activity objective or teacher scope. Check participation, practical judgment, and reflection against that local scope. |
| Technology/making | Arduino, robotics, electronics, maker topics | Check component function, restricted code or state tracing, wiring interpretation, debugging, safe procedure, equipment/version conditions, and prohibited-risk boundaries. |
| Coding | Non-curriculum coding topics | Check code trace, algorithm, state change, debugging, representation, runtime assumptions, and versioned APIs. |
| Document comprehension | Any document-grounded custom topic | Make every answer follow from selected chunks, distinguish direct support from inference, and block unsupported or conflicting claims. |

For grades 1–2, reject avoidable multi-clause prompts, unexplained technical vocabulary, and choices whose decoding load exceeds the intended concept.
